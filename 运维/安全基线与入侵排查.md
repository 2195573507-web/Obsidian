# 安全基线与入侵排查

> 关联：[[运维/服务器安全加固、备份、监控与应急响应|服务器安全加固、备份、监控与应急响应]]、[[server/服务器 FAQ 与故障排查手册|服务器 FAQ 与故障排查手册]]

## 1. 安全基线

- SSH 密钥登录，禁用密码或限制；
- UFW 仅开放必要端口；
- 数据库不暴露公网；
- Nginx 统一入口；
- 服务最小权限用户运行；
- .env 权限 600；
- 日志脱敏；
- 定期备份和恢复演练；
- systemd/docker 异常有告警。

## 2. 入侵排查顺序

不要先删文件。先保留证据：登录、进程、网络、定时任务、systemd、文件变更、SSH key、用户列表、bash history。

```bash
who
last -a | head -100
ps auxf
ss -tulpn
systemctl list-units --type=service --state=running
crontab -l
find /root /home /tmp /var/tmp -mtime -3 -type f -ls 2>/dev/null
```

## 3. SSH 异常

大量 failed 不等于入侵，未知 Accepted 才高危。检查 authorized_keys、sudo 用户、最近新增用户、登录来源。

## 4. 可疑进程

关注高 CPU、未知路径、/tmp 执行、随机名、外连矿池、隐藏进程。不要立即 kill，先记录 pid、cmdline、cwd、exe、网络连接。

## 5. 处置

轻微扫描：加固 SSH 和 fail2ban。疑似入侵：隔离网络、轮换密钥、保留镜像、从干净备份恢复。手工清毒不如重装可靠。
