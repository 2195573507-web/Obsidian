# 发布变更、回滚与容量规划

> 关联：[[运维/服务器运维 MOC|服务器运维 MOC]]、[[软件工程/测试与质量保障体系|测试与质量保障体系]]

## 1. 发布原则

小步变更、可观测、可回滚、一次只改一个关键变量。上线前记录当前状态，备份配置，明确成功/失败标准。

## 2. 发布清单

- [ ] 变更目标明确；
- [ ] 影响域名/端口/服务明确；
- [ ] 配置和数据已备份；
- [ ] 回滚命令已写好；
- [ ] 健康检查命令已准备；
- [ ] 低峰期或维护窗口；
- [ ] 监控和日志已打开。

## 3. 回滚策略

代码回滚、配置回滚、数据库回滚、流量回滚、证书回滚、容器镜像回滚分别设计。数据库破坏性迁移不能只靠代码回滚。

## 4. 容量规划

关注 CPU 峰值、内存 available、磁盘增长率、inode、Docker overlay、数据库大小、日志增长、备份空间、带宽、证书和域名数量。

## 5. 压测与观察

压测前确认授权和范围。观察 QPS、P95、错误率、CPU、内存、IO、DB 慢查询、Nginx upstream time。压测后清理临时数据并记录容量结论。
