# 备份恢复演练

> 关联：[[运维/服务器安全加固、备份、监控与应急响应|服务器安全加固、备份、监控与应急响应]]

## 1. 核心原则

没有恢复演练的备份不算备份。备份体系要回答：备什么、多久备、备到哪、保留多久、谁能恢复、恢复需要多久、恢复后如何验证。

## 2. 备份对象

数据库、上传文件、配置、.env、Nginx、systemd、Docker compose、Docker volume、Obsidian vault、Agent memory、证书配置、脚本。

## 3. RPO/RTO

RPO：最多能丢多少数据。RTO：多久恢复服务。不同服务分级制定，不能所有服务同标准。

## 4. 演练流程

1. 选取备份点；2. 在临时目录/临时库恢复；3. 校验文件数量、checksum、数据库表数、核心查询；4. 启动临时服务；5. 记录耗时；6. 修复文档缺口。

## 5. SQLite

在线备份用 `.backup`，不要直接复制写入中的 db。WAL 文件也要考虑。

## 6. Postgres

逻辑备份：`pg_dump -Fc`；恢复测试：`createdb restore_test && pg_restore -d restore_test file.dump`。大库需要物理备份和 WAL 归档。

## 7. 恢复验收

- 服务能启动；
- 核心 API 返回 200；
- 数据量符合预期；
- 最近数据存在；
- 权限和密钥未泄露；
- 日志无异常；
- 用户上传文件可访问。
