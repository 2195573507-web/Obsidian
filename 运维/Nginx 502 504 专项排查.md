# Nginx 502 504 专项排查

> 关联：[[运维/Nginx 反向代理与 HTTPS 运维|Nginx 反向代理与 HTTPS 运维]]

## 502

含义：Nginx 连不上 upstream。检查：

```bash
sudo tail -n 100 /var/log/nginx/error.log
sudo nginx -T 2>&1 | grep -E 'server_name|proxy_pass'
ss -tulpn
systemctl --failed --no-pager
docker ps -a
```

## 504

含义：upstream 超时。查后端卡死、数据库慢查询、外部 API、proxy_read_timeout。

## 不要做

不要直接重启所有服务；不要改多个 proxy_pass；不要忽略最近部署。
