# 证书续期故障排查

> 关联：[[运维/Nginx 反向代理与 HTTPS 运维|Nginx 反向代理与 HTTPS 运维]]

## 1. 检查

```bash
sudo certbot certificates
systemctl list-timers --all | grep -i certbot || true
echo | openssl s_client -servername example.com -connect example.com:443 2>/dev/null | openssl x509 -noout -dates -issuer -subject
```

## 2. 常见原因

DNS 指错、80/443 不通、Nginx server_name 不匹配、ACME 路径被跳转破坏、证书 SAN 不包含域名、续期后未 reload。

## 3. 处理顺序

验证 DNS → 验证 80/443 → `certbot renew --dry-run` → 看日志 → 修 Nginx → reload。真实续期和新增证书需确认。
