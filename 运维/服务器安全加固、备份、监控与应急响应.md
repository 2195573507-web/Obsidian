# 服务器安全加固、备份、监控与应急响应

> 关联：[[server/服务器 FAQ 与故障排查手册|服务器 FAQ 与故障排查手册]]、[[运维/Linux 系统管理与故障排查 SOP|Linux 系统管理与故障排查 SOP]]

## 1. 安全原则

最小暴露面、最小权限、密钥不进公开仓库、日志可追踪、备份可恢复、变更可回滚、事故能止血。

## 2. SSH

建议：密钥登录、禁用密码、禁止 root 密码登录、fail2ban、限制管理 IP。修改 SSH 前保留当前会话并确认备用入口。

只读检查：

```bash
systemctl status --no-pager ssh
ss -tulpn | grep ':22'
last -a | head -50
journalctl -u ssh --since '24 hours ago' --no-pager | grep -Ei 'Failed|Accepted|Invalid' | tail -100
```

## 3. 防火墙

公网通常只开放 22/80/443。内部服务优先 127.0.0.1 或 Docker 内网。UFW reset、删除 22、开放数据库端口都是高风险。

## 4. 备份

备份对象：数据库、上传文件、配置、.env、Docker volumes、Obsidian vault、Agent memory。备份成功不等于可恢复，必须恢复演练。

SQLite 用 `.backup`，Postgres 用 `pg_dump -Fc`，文件用 tar + sha256。密钥类备份要加密。

## 5. 监控

至少监控：HTTP 200、证书到期、磁盘、inode、内存、load、Docker health、systemd failed、SSH failed/accepted、备份成功、Nginx 5xx。

## 6. 应急响应

顺序：止血 → 保留证据 → 恢复服务 → 查根因 → 防复发。疑似入侵不要先清日志。磁盘满不要先删未知大目录。服务挂不要先重启全机器。

事故记录应包含：时间线、影响范围、现象、临时修复、根因、永久修复、后续行动。
