# 2026-06-23 OpenClaw记忆质量巡检

## 结论
- 本次巡检发现 5 类待确认问题：2 组路由记忆冲突/重复、1 组旧自动学习任务残留、1 组共享导出健康状态不一致、1 组低价值 temporary 噪声线索。
- 未执行任何删除、禁用或覆盖写入；仅输出建议动作与待确认清单。

## 记忆检索健康
- `shared-agent-memory`：在运行（`active`）。
- `shared-local-task-worker`：未运行（`inactive`），与“已启用后台 worker”的旧记忆存在不一致，需确认当前预期。
- `local_tasks` / `/opt/hermes-learning`：目录存在，说明旧自动学习产物或痕迹仍在；是否应继续保留需按当前策略确认。
- 向量索引：`memory_search` 可返回高相关结果，说明检索可用；但命中中仍存在明显过期/冲突项，需做知识清理。

## 待确认问题清单
1. **旧 BJT 03:00-05:00 自动学习任务**
   - 状态：已过期/疑似废弃，但库中仍有“已创建”与“未恢复”两类记录并存。
   - 证据摘要：一条旧创建记录指向 `/opt/hermes-learning/YYYY-MM-DD/`，另一条后续记录明确写“未恢复”。
   - 建议动作：`expire` 旧创建记录；对后续“未恢复”记录保留为历史说明或 `rewrite` 为“已废弃”。
   - 处理级别：**需要用户确认**。

2. **local-first 作为全局前置路由**
   - 状态：存在至少 3 条互相冲突的记忆：一条说“默认关闭/已取代”，一条说“已启用”，另一条说“由轻量任务池取代”。
   - 风险：会误导后续任务以为 OpenClaw 仍依赖全局前置 local-first。
   - 建议动作：保留最新结论为 `keep`，其余旧表述 `expire` 或 `rewrite` 成“历史实现/已回滚”；必要时 `merge` 成单一事实页。
   - 处理级别：**需要用户确认**。

3. **shared-agent-memory / local-task-worker 健康描述冲突**
   - 状态：当前系统显示主服务 active、worker inactive，但记忆里仍写“已启用后台 worker 和保温 timer”。
   - 风险：对巡检和运维判断有误导。
   - 建议动作：`rewrite` 为“历史状态记录”或补充当前状态；如 worker 确认已停用，则 `expire` 旧描述。
   - 处理级别：**需要用户确认**。

4. **低价值 temporary / failure / noise 项**
   - 状态：检索中出现多条 `temporary/openclaw` 迁移残留和重复上下文（如用户信息、基础环境说明、一次性迁移痕迹）。
   - 风险：噪声较高，降低检索精度。
   - 建议动作：对纯环境/迁移痕迹做 `expire`，保留真正稳定的用户偏好与系统事实；重复项优先 `merge`。
   - 处理级别：建议批量人工复核后再处理。

5. **重复/近重复记忆**
   - 状态：`model-routing`、`local-first-routing`、`local-task-pool` 三组内容高度重叠。
   - 建议动作：统一为一条“当前路由策略”主记忆，其余拆成历史变更记忆并标注时间线。
   - 处理级别：**需要用户确认**。

## 建议动作汇总
- `keep`：当前有效的 shared-memory 服务可用、主模型回退、最新路由策略结论。
- `disable`：本次不建议自动执行，除非用户后续明确授权。
- `expire`：旧 BJT 03:00-05:00 自动学习任务的“已创建”表述、过期路由表述、纯迁移噪声。
- `merge`：多条 local-first / model-routing 记忆合并成单一事实页。
- `rewrite`：将“已启用”但过时的 worker/路由表述改为历史状态或当前状态说明。

## 备注
- 本次仅做巡检，不做删除/禁用。
- 若后续要落地清理，建议先按“当前有效事实 / 历史变更 / 一次性噪声”三类分层处理。
