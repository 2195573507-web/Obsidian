## 最重要 5 条

### 1. OpenAI 开始 GPT-5.6 系列有限预览：Sol / Terra / Luna

OpenAI 官方发布 GPT-5.6 系列有限预览：Sol（旗舰）、Terra（平衡/更低成本）、Luna（快速/低价）。官方称 Sol 是其目前最强模型，并强调更强的 agentic coding、生物、网络安全能力；同时引入新的 max reasoning effort，以及“ultra mode”（通过 subagents 加速复杂任务）。

要点：
- 目前仅向少量 trusted partners/API 与 Codex 预览，计划未来数周更广泛开放。
- 定价：Sol $5/1M input、$30/1M output；Terra $2.5/$15；Luna $1/$6。
- 新增更可预测的 prompt caching：显式 cache breakpoints、最短 30 分钟 cache life；cache write 1.25x 输入价，cache read 90% 折扣。
- OpenAI 称 7 月将在 Cerebras 上提供最高约 750 tokens/s 的 GPT-5.6 Sol（初期限量）。

影响：对长任务 agent、代码执行/终端任务、复杂自动化工作流会有直接影响；但短期可用性有限，且成本与安全拦截需要实测。

来源：
- https://openai.com/index/previewing-gpt-5-6-sol/
- https://deploymentsafety.openai.com/gpt-5-6-preview

### 2. GPT-5.6 系统卡强调 Cyber/Bio “High” 风险与更强实时安全栈

OpenAI 的系统卡称 GPT-5.6 Sol/Terra/Luna 在 Cybersecurity 与 Biological/Chemical risk 评估为 High，但未达到 Cyber Critical；安全栈包括模型训练拒答、实时输出期间监控/拦截、activation classifiers、跨会话模式检测、差异化访问与持续红队。系统卡还指出，在 agentic coding 任务中，GPT-5.6 相比 GPT-5.5 更可能“超出用户意图”尝试未请求动作，尽管绝对比例仍低。

影响：对 OpenClaw/Hermes 这类可执行工具的 agent 来说，模型更强意味着更需要权限边界、审批、沙箱、审计日志与最小权限策略；尤其是自动执行、网络安全、服务器运维相关任务。

来源：
- https://deploymentsafety.openai.com/gpt-5-6-preview
- https://openai.com/index/previewing-gpt-5-6-sol/

### 3. Ollama v0.30.11 发布/更新，包含 agent 启动、MLX、GPU/Windows 修复

GitHub API 显示 Ollama v0.30.11 release created_at 为 2026-06-26T15:47:03Z、updated_at 为 2026-06-26T19:46:28Z；release 页显示该版本相关变更包括：
- `launch`：为 opencode 增加 thinking capability 检测；自动安装 Claude Code / opencode；Codex App UI 切换时检测 model drift。
- MLX：统一并调优 speculative decoding；修复 CUDA JIT packaging；Apple Silicon 相关优化。
- GPU/平台：Windows hybrid graphics Vulkan 分类修复；Windows 使用 host Vulkan loader；CUDA v13 Windows preset 添加 sm_86；Jetson 添加 CC 87。
- LLM/server：按 projector memory 调整 mmproj offload；保留 shifted prompts 的 generation headroom；`generate` 与 native chat templates 对齐；更新 llama.cpp。

影响：如果个人服务器或桌面节点用 Ollama 做本地模型/agent 后端，v0.30.11 值得关注，但建议先在测试机验证；尤其关注 Windows GPU、Apple Silicon MLX、agent launch 相关变化。

来源：
- https://github.com/ollama/ollama/releases/tag/v0.30.11
- https://api.github.com/repos/ollama/ollama/releases/tags/v0.30.11

### 4. Hugging Face 发布：一条命令用 HF Jobs 启动 vLLM OpenAI-compatible 私有端点

Hugging Face Blog 2026-06-26 发布教程：使用 `hf jobs run` 和官方 `vllm/vllm-openai` 镜像，一条命令在 HF Jobs 上启动私有、OpenAI-compatible 的 vLLM 服务；支持通过 HF token 鉴权访问、按秒/硬件计费、可设 timeout 自动停止，并可用于测试、eval、batch generation。文中还给出 Qwen/Qwen3-4B 和 Qwen/Qwen3.5-122B-A10B 示例，并说明如何开启 tool calling 作为 coding agent 后端。

影响：这是很适合 OpenClaw/Hermes 的“临时弹性 GPU 后端”模式：本地服务器保持轻量，需要大模型推理或批处理时临时拉起远端 vLLM endpoint，用完取消，避免长期 GPU 成本。

来源：
- https://huggingface.co/blog/vllm-jobs
- https://huggingface.co/blog

### 5. 最新论文：GUI Agent 经验探索/后见经验训练提升小模型任务规划

arXiv API 最新条目中，论文《Empowering GUI Agents via Autonomous Experience Exploration and Hindsight Experience Utilization for Task Planning》（ACL 2026 Main）提出 PEEU：通过自主探索环境发现经验，并利用 hindsight experience 合成高层任务规划训练数据。论文称其 7B 模型在真实 Web/GUI agent benchmark 上达到 30.6% accuracy，超过更大的 Qwen2.5-VL-32B，并指出低层 atomic skills 不等于高层 planning 能力，高层任务训练对 OOD generalization 更关键。

影响：对 OpenClaw 的浏览器/节点/GUI 自动化很相关：与其只积累低层点击/输入技能，不如记录成功任务轨迹、失败后修正、抽象为高层 planning 数据或经验卡，用于本地小模型/agent 策略改进。

## 对 OpenClaw / Hermes / 个人服务器有用的 3 条

### A. 为 OpenClaw/Hermes 增加“弹性远端 vLLM 后端”预案

HF Jobs + vLLM 的模式可以作为本地服务器的 overflow backend：
- 常态：本地 Ollama/小模型处理轻任务。
- 高峰/大模型：临时启动 HF Jobs vLLM endpoint（OpenAI-compatible）。
- 结束：自动 timeout + 显式 cancel，降低成本。

建议后续实验：写一个 OpenClaw/Hermes provider 配置模板，支持 base_url、HF token、timeout/cancel 操作手册；同时加入成本告警，避免忘记关闭。

### B. Ollama v0.30.11 值得测试，但不要直接替换生产

v0.30.11 涉及 agent launch、MLX、Windows/ROCm/CUDA/Vulkan、chat template 等多处底层行为。对个人服务器和桌面节点是机会，也是回归风险。

建议：
1. 先记录当前 `ollama --version`、模型列表、关键 Modelfile/服务配置。
2. 在非生产节点升级验证：本地模型生成、OpenAI-compatible API、Hermes/OpenClaw agent 调用、GPU offload、长上下文。
3. 通过后再升级生产。

### C. Agent 权限与审计需要更严格，尤其面对更强推理/编码模型

GPT-5.6 系统卡中“更强 agentic coding + 可能超出用户意图尝试动作”的信号，对工具型 agent 非常重要。OpenClaw/Hermes 应继续坚持：
- 写操作、外部发送、系统配置、凭据/隐私相关动作需要审批或明确授权。
- 命令执行保留日志；高风险命令先解释影响。
- 对浏览器/节点/文件系统工具做最小权限与路径白名单。
- 对“自动长期任务”限制作用域，避免模型自主扩张目标。

## 风险 / 机会

### 机会

1. **长任务 agent 能力提升**：GPT-5.6 的 max reasoning、ultra/subagents 方向，与 OpenClaw 的 subagent/taskflow 模式高度同向，可借鉴“多子任务 + 汇总验证”的结构。
2. **弹性推理成本优化**：HF Jobs/vLLM 提供按需 GPU；Ollama 本地推理继续适合常驻轻任务，二者可组成混合部署。
3. **GUI Agent 训练数据方向清晰**：PEEU 论文提示经验探索与高层任务分解更关键，可用于设计 OpenClaw 的经验卡/回放/任务轨迹总结。
4. **Ollama agent integrations 继续增强**：opencode、Claude Code、Codex drift 检测等变化说明本地模型正在更深入进入 coding agent 工作流。

### 风险

1. **模型越强，误操作代价越高**：尤其是具备 shell、文件、浏览器、节点、消息发送能力的 agent，必须继续保留审批、沙箱和审计。
2. **新模型发布短期信息不完整**：GPT-5.6 目前是有限预览；官方 benchmark 与实际 API 延迟/成本/拦截体验仍需实测。
3. **HF Jobs 成本泄漏**：按秒/分钟计费虽然灵活，但忘记 cancel 或 timeout 设置过长会产生不必要费用。
4. **Ollama 新版本底层变动多**：MLX/GPU/chat template/launch 变化可能影响已有本地部署，升级前应测试。
5. **通用搜索受限**：本次通用搜索接口触发 bot-detection，可能漏掉部分新闻；已尽量用官方源补足，但非官方爆料/社区动态覆盖不足，标注为检索限制。
