## 最重要 5 条

### 1. CISA KEV 新增 Microsoft SharePoint Server CVE-2026-45659（2026-07-01）

- **来源可靠性**：高，CISA KEV 官方 JSON Feed。
- **要点**：CISA KEV 目录版本 `2026.07.01`，发布时间 `2026-07-01T19:00:06Z`。新增 `CVE-2026-45659`，Microsoft SharePoint Server “反序列化不可信数据”漏洞，描述为授权攻击者可通过网络执行代码；CISA 给出的处置期限为 `2026-07-04`。
- **影响面**：如果个人/公司服务器暴露 SharePoint 或相关集成服务，应按 KEV 优先级处理。对 OpenClaw/Hermes 本体通常不直接相关，但若服务器上有企业协作组件或反向代理到 SharePoint，需要立即排查。
- **建议**：资产清点；确认是否运行 SharePoint；如有，按 MSRC/CISA 指引修补或缓解，并保留取证日志。
- **来源**：
  - https://www.cisa.gov/sites/default/files/feeds/known_exploited_vulnerabilities.json
  - https://www.cisa.gov/known-exploited-vulnerabilities-catalog

### 2. SimpleHelp OIDC 认证绕过仍处于 KEV 紧急窗口（CVE-2026-48558，due 2026-07-02）

- **来源可靠性**：高，CISA KEV 官方 JSON Feed。
- **要点**：`CVE-2026-48558` 于 `2026-06-29` 加入 KEV，漏洞点是 SimpleHelp 在 OIDC 登录流程中未验证 identity token 的加密签名；在脆弱配置下，远程未认证攻击者可伪造身份声明获得 technician session，部分配置可绕过 MFA。CISA 处置期限为 `2026-07-02`。
- **影响面**：远程支持/运维工具一旦被接管，通常等同服务器入口被接管。即使不是 Linux 漏洞，也与个人服务器运维风险高度相关。
- **建议**：如使用 SimpleHelp，立刻升级/缓解；检查异常 technician 登录、远程会话、账户变更和 OIDC 配置。
- **来源**：
  - https://www.cisa.gov/sites/default/files/feeds/known_exploited_vulnerabilities.json
  - https://simple-help.com/security/simplehelp-security-update-2026-05

### 3. runc 三个高危容器逃逸漏洞仍需核查：CVE-2025-31133 / CVE-2025-52565 / CVE-2025-52881（非 24h 新披露，但近期仍被安全媒体和云厂商引用）

- **来源可靠性**：高，runc/GitHub 官方 Release 与 Advisory、oss-sec；媒体报道为辅助。
- **要点**：三个漏洞可通过不同方式绕过 runc 对 `/proc` 写入限制，造成容器逃逸、宿主机 root 权限或拒绝服务。受影响环境包括 Docker、containerd、CRI-O、Kubernetes 以及云上托管 K8s 的工作节点。修复版本包括 `runc 1.2.8`、`1.3.3`、`1.4.0-rc.3` 及之后版本。
- **影响面**：对 OpenClaw/Hermes/个人服务器非常相关：如果服务以 Docker/K8s 部署，且允许构建或运行不可信镜像/Dockerfile，风险显著上升。
- **建议**：检查 `runc --version`、Docker/containerd 包版本；升级后重启 Docker/containerd 或轮换 K8s 节点；避免运行不可信镜像，启用镜像签名/准入控制/运行时检测。
- **来源**：

  - https://github.com/opencontainers/runc/releases/tag/v1.2.8
  - https://github.com/opencontainers/runc/security/advisories/GHSA-9493-h29p-rfm2
  - https://github.com/opencontainers/runc/security/advisories/GHSA-qw9x-cqr3-wc7r
  - https://github.com/opencontainers/runc/security/advisories/GHSA-cgrx-mc8f-2prm

  - https://www.bleepingcomputer.com/news/security/dangerous-runc-flaws-could-allow-hackers-to-escape-docker-containers/

### 4. AWS 对 runc 漏洞的云服务侧响应：托管服务多由平台处理，自管理节点仍需用户动作（非 24h 新披露）

- **来源可靠性**：高，AWS Security Bulletin。
- **要点**：AWS Bulletin `AWS-2025-024` 说明 AWS 不把容器作为客户间安全边界，整体无跨客户风险；但使用容器隔离自管工作负载的客户应更新 OS/vendor 包。Amazon Linux、Bottlerocket、ECS EC2、EKS 节点等均有对应 AMI/包更新说明。EKS 上仅升级控制面不够，节点镜像也要更新/轮换。
- **影响面**：如果个人服务器在 AWS EC2/EKS/ECS 上跑 OpenClaw/Hermes，重点不是“控制面版本”，而是底层节点 AMI 和 container runtime。
- **建议**：云上自管节点做 AMI/包升级；托管节点组触发滚动更新；Karpenter/自管节点检查 drift/AMI selector。
- **来源**：
  - https://aws.amazon.com/security/security-bulletins/rss/aws-2025-024/

## 对 OpenClaw / Hermes / 个人服务器有用的 3 条

1. **优先排查容器运行时版本**  
   - 如果 OpenClaw/Hermes 使用 Docker 或 containerd 部署，检查 `runc`、Docker、containerd 的发行版补丁版本。漏洞修复不一定表现为上游 `1.3.3` 版本号，发行版可能 backport，需要结合包 changelog/安全公告确认。

2. **远程运维入口比业务服务更危险**  
   - SimpleHelp 这类远程支持工具的认证绕过进入 KEV，提醒要把“远程管理面板、监控面板、自动化任务面板、CI/CD、Webhook 服务”列为最高优先级资产。建议：面板只走 VPN/Tailscale/Zero Trust；启用 MFA；保留登录审计；关闭公网直连。

3. **K8s/容器安全不要只看控制面版本**  
   - runc 事件再次说明：工作节点、AMI、runtime、镜像供应链、运行时检测都重要。个人服务器也应采用“只跑可信镜像、最小特权、rootless/用户命名空间、只读根文件系统、限制 Docker socket 暴露”的基线。

## 风险 / 机会

### 风险

- **容器逃逸风险**：如果能运行不可信镜像或构建不可信 Dockerfile，runc 漏洞链可能导致宿主机被写入 `/proc/sys/kernel/core_pattern` 等关键文件，进而实现宿主机级影响。
- **远程运维工具成为入口**：SimpleHelp 这类工具一旦认证绕过，可能直接绕过常规边界防护。
- **“托管云服务已修”误判**：云厂商处理控制面/托管服务，不等于你的自管节点、AMI、DaemonSet、容器镜像全部已修。
- **来源不足风险**：搜索中出现了一些非官方站点声称 Docker 2026 新 CVE，但未能从 Docker 官方公告或可靠数据库在 24h 窗口内确认，暂不采纳为事实。

### 机会

- **建立每日自动巡检清单**：KEV 新增项、Kubernetes CVE Feed、Docker/runc/containerd Release、云厂商安全公告可做成每日自动任务。
- **强化供应链**：为 OpenClaw/Hermes 镜像加入 SBOM、镜像签名、固定 digest、定期重建基础镜像。
- **增加运行时检测**：Falco/Sysdig 类规则可监控异常 symlink、对 `/proc/sysrq-trigger`、`/proc/sys/kernel/core_pattern` 的异常访问，适合作为个人服务器轻量告警项。

## 今日建议动作

- [ ] 检查服务器是否运行 SharePoint、SimpleHelp 或相关远程运维工具；如有，按 CISA KEV 优先级处理。
- [ ] 在 Docker/K8s 节点上核查 `runc`/Docker/containerd 包版本与发行版安全公告。
- [ ] 若使用 EKS/AKS/GKE 或云主机 AMI，确认工作节点镜像是否已轮换到包含运行时补丁的版本。
- [ ] 审计是否有容器挂载 Docker socket、特权容器、hostPID/hostNetwork、可写宿主路径。
- [ ] 对公网管理入口增加 IP allowlist/VPN/Zero Trust，避免直接暴露。
