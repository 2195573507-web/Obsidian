### 5. Docker Engine 高危 AuthZ 绕过漏洞（CVE-2026-34040，CVSS 8.8）

新发现的高危漏洞允许攻击者通过构造超 1MB 的填充 HTTP 请求绕过 Docker 授权插件（AuthZ），创建特权容器并获得宿主机 root 访问权限。这是 CVE-2024-41110 的不完整修复导致的新问题。攻击者无需复杂利用代码，一个简单填充 HTTP 请求即可触发。

影响范围：所有依赖 Docker AuthZ 插件的云和 Kubernetes 环境。可导致 AWS 凭证、SSH 密钥、K8s 配置文件泄露。

📎 https://www.cyasha.com/cve-2026-34040-docker-cloud-and-kubernetes/

## 🛠 对 OpenClaw / Hermes / 个人服务器有用的 3 条

### 1. 🐳 Docker Engine AuthZ 绕过（CVE-2026-34040）— 紧急自查

如果你的个人服务器或 OpenClaw/Hermes 运行在 Docker 容器中，**立即检查 Docker 版本**。该漏洞允许容器逃逸和宿主机完全控制。受影响的 Docker Desktop 版本需更新至 4.71.0+。

**行动**：运行 `docker version` 确认当前版本，参考 Docker 官方安全公告更新。

📎 https://docs.docker.com/security/security-announcements/

### 2. 🐧 Linux 内核本地提权漏洞连环爆发 — 检查内核版本

近期多个高危 Linux 内核提权漏洞集中曝光，均已有公开 PoC：

- **CVE-2026-31431 "Copy Fail"** — AF_ALG 加密子系统，732 字节 Python 脚本即可提权，影响 2017 年以来所有主流发行版。CISA KEV 收录。
- **CVE-2026-23111** — nf_tables 中一个感叹号错误导致 use-after-free，Debian/Ubuntu 上可稳定提权（>99% 成功率）。
- **CVE-2026-47337** — Ubuntu 内核 AppArmor 网络套接字空指针解引用。

**行动**：`uname -r` 查看内核版本；修复版本为 6.18.22 / 6.19.12 / 7.0。

📎 https://arstechnica.com/security/2026/06/a-single-errant-character-in-the-linux-kernel-allows-attacker-to-gain-root/
