   b9837 新增 `--reasoning-preserve`，b9835 修复 UI 单模型模式下 stop 与 reasoning skip 行为。对接入 Qwen、DeepSeek、Llama 等 reasoning 模型的本地 Agent 很关键：升级可能改变 reasoning 展示、停止条件、模板渲染与结构化输出表现。

2. **Agent 工具链安全边界问题变得更现实：敏感文件 ignore / denylist 应上升为运行时能力**  
   开源报告提到 OpenAI Codex issue #2847 仍有用户请求 `.codexignore` / 全局 ignore，用于禁止 Agent 读取或发送 `.env`、`.pem`、`.ssh/`、`.aws/` 等路径。这说明“靠提示词要求 Agent 不读密钥”不够，应由工具层强制。  
   来源：`/root/obsidian-vault/资讯更新/开源与开发者工具/2026-06-29-OpenClaw.md`；<https://github.com/openai/codex/issues/2847>

## 对 OpenClaw / Hermes / 个人服务器有用的 3 条

1. **给模型后端升级加最小回归测试矩阵**  
   建议覆盖：reasoning 标签是否泄露/丢失/重复、stop token 是否生效、tool call/JSON 输出是否稳定、首 token 延迟、流式 chunk gap、本地模型与云模型 fallback 行为。llama.cpp 今日变更正好提示这类测试很必要。

2. **把敏感路径保护做成默认安全边界**  
   对 OpenClaw/Hermes/coding agent，建议默认保护 `.env*`、SSH 私钥、云凭证、浏览器 profile、备份、token 缓存、生产配置等路径；读取或上传需显式审批或工具层拒绝。此项比单纯“提醒模型不要泄露”更可靠。

3. **个人服务器安全优先级：内核/容器/边界设备/备份快照**  
   今日材料共同指向四个动作：检查内核安全更新与待重启状态；更新 Docker/containerd/runc/BuildKit 链路；若用 K8s，收紧 Pod/PV/Ingress 创建权限；对 NAS/SMB/NFS 备份目录启用快照或版本化，降低路径穿越/误删损失。

## 风险 / 机会

### 风险

- **本地提权与容器逃逸组合风险**：DirtyClone、内核、containerd、CSI、ingress-nginx 等问题说明低权限执行点仍可能扩展到宿主机或集群 Secret。
- **Agent 密钥暴露风险**：编码 Agent 如果能读 repo 外敏感文件或上传上下文，`.env`、SSH、云凭证、浏览器 profile 都可能成为事故入口。
- **一键安装与安全 Agent 双刃剑风险**：MCP、Pentest Agent、Bash-first LLM CLI 都可能修改配置、读取密钥或发起网络测试，必须隔离验证。
- **来源覆盖不足风险**：本次基于当天已有报告综合，未额外联网；未覆盖社交媒体、Discord、全部厂商公告。未能确认的内容已不纳入核心结论或标注未确认。

### 机会

- **建立 OpenClaw/Hermes “模型后端健康检查”**：将 reasoning、tool call、JSON、延迟、fallback、成本与错误归因做成定期巡检。
- **建设轻量 AI Gateway 治理面**：先做 request_id、虚拟 key metadata、provider/model routing、fallback 计数、429/5xx 分类与脱敏日志，再考虑完整 Grafana/OTel。
- **把安全检查前移到脚本和配置层**：Shell、Dockerfile、systemd、nginx、compose、cron、CI 脚本应纳入扫描与人工审批流程。
