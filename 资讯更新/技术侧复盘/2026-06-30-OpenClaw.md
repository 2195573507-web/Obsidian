## 最重要 5 条

1. **Agent 运行时安全正在从“提示词防护”升级为“免疫层/治理层”**
   - ANIS 论文、Adversa AI 月度汇总与多起 agentic AI 风险都指向同一趋势：持久记忆、工具调用、子 Agent、CI/代码执行链路会引入记忆投毒、工具劫持、符号链接/路径逃逸、跨会话污染等运行时风险。
   - 对 OpenClaw/Hermes：需要把外部内容标记为不可信输入，给工具调用、记忆写入、文件写入、子任务通信增加审计与隔离。
   - 来源：`/root/obsidian-vault/资讯更新/AI与大模型/2026-06-30-OpenClaw.md`；`/root/obsidian-vault/资讯更新/技术晚间补充/2026-06-30-OpenClaw.md`；https://arxiv.org/abs/2606.28270

2. **远程运维与身份集成风险优先级升高：SimpleHelp OIDC 认证绕过已进 CISA KEV**
   - CISA 2026-06-29 将 SimpleHelp CVE-2026-48558 加入 KEV；脆弱 OIDC 配置下，远程未认证攻击者可能伪造身份令牌取得 technician 会话，部分配置可能绕过 MFA。
   - 对个人服务器：所有远控/管理面/SSO/OIDC/JWT 集成应优先核查签名、issuer、audience、过期时间、nonce/state、MFA 绑定与访问日志。
   - 来源：`/root/obsidian-vault/资讯更新/服务器安全运维/2026-06-30-OpenClaw.md`；`/root/obsidian-vault/资讯更新/技术晚间补充/2026-06-30-OpenClaw.md`；https://www.cisa.gov/known-exploited-vulnerabilities-catalog

3. **容器运行时与 Docker API/授权面值得重点排查**
   - containerd CRI 插件多项漏洞影响 K8s/ECS/EKS/Fargate/自管节点；Docker Engine AuthZ 绕过研究披露称大于 1MB 请求体可能绕过授权插件（本条官方确认不足，标注未确认）。
   - 对个人服务器：禁止公网暴露 Docker API；避免把 `/var/run/docker.sock` 暴露给容器；不跑不可信镜像；跟进 containerd、Docker Engine、内核与基础镜像修复。
   - 来源：`/root/obsidian-vault/资讯更新/服务器安全运维/2026-06-30-OpenClaw.md`；`/root/obsidian-vault/资讯更新/技术晚间补充/2026-06-30-OpenClaw.md`；https://aws.amazon.com/security/security-bulletins/2026-046-aws/；Docker AuthZ 研究未确认：https://www.cyera.com/research/one-megabyte-to-root-how-a-size-check-broke-dockers-last-line-of-defense

4. **本地/混合模型与 OpenAI-compatible 路由继续成为现实路线**

   - 对 OpenClaw/Hermes：保持 OpenAI-compatible 抽象、model alias/group、fallback、成本/速率控制，比绑定某个 provider 更稳。

   - 今日项目学习报告建议：下一阶段重点应放在 LLM/API 流式链路的可诊断、可限流、可回滚、可解释，包括 first-token latency、active streams、chunk gap、fallback/rate-limit、provider/model health、结构化错误与成本追踪。

## 对 OpenClaw / Hermes / 个人服务器有用的 3 条

1. **建立 Agent 安全基线清单**
   - 默认外部网页、仓库、Issue、日志、聊天消息均为不可信输入。
   - 对 `exec`、文件写入、网络发送、配置变更、记忆写入、子 Agent 指令传递保持最小权限与审计。
   - 将符号链接、路径穿越、隐藏文件、工作区外写入、工具输出注入列为高风险检查项。

2. **把个人服务器控制面做一次轻量盘点**
   - 优先级：KEV/已利用漏洞 > 公网管理入口 > Docker/containerd/内核 > 基础工具链 > 自动化脚本依赖。
   - 今日可检查：是否使用 SimpleHelp/类似远控；Docker API 是否外露；容器是否挂载 docker.sock；containerd/Docker/kernel 是否等待补丁；备份脚本是否连接不可信 SSH/SFTP。

3. **把 LLM Gateway 设计成治理平面**
   - 推荐能力：virtual key、model alias/group、provider health、fallback/circuit breaker、rate limit/budget、结构化日志、first-token latency、stream disconnect、成本估算。
   - 避免把 request_id、完整 URL、用户 ID、密钥 hash、prompt 放进 Prometheus label；敏感内容只做脱敏日志或不记录。

## 风险 / 机会

### 风险

- **Agent 执行链跨边界风险扩大**：提示注入可能影响工具调用、文件系统、CI、记忆与后续任务，单靠 prompt 规则不够。
- **远程运维入口被利用窗口短**：SimpleHelp KEV 说明远控/管理面一旦认证绕过，会快速进入攻击优先级。
- **容器与本地提权组合风险高**：旧内核、containerd/Docker 漏洞、docker.sock 暴露、特权容器会把单点入侵放大成主机级风险。
- **开源模型/热榜项目存在未确认夸大**：Ornith、Qwen 本地体验、Docker AuthZ 研究、DirtyClone 等条目需等待第三方复测或官方公告，不宜直接生产替换。

### 机会

- **OpenClaw/Hermes 可形成“安全型 Agent 平台”差异化**：工具策略、记忆审计、任务隔离、回放、来源标注都是可落地能力。
- **混合推理能降低成本并提升隐私**：本地小模型做监控/摘要/预处理，云端或临时 GPU 只处理高价值困难任务。
- **监控指标可快速转化为运维收益**：first-token latency、active streams、fallback、429、timeout、provider health 能直接定位 LLM 链路问题。
