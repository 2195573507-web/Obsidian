## 最重要 5 条

1. **runc 三个高危容器逃逸漏洞（确认影响所有已知版本）**
   - CVE-2025-31133 / CVE-2025-52565 / CVE-2025-52881，均涉及 `/proc` 写入限制绕过，影响所有已知 runc 版本。升级到 1.2.8 / 1.3.3 / 1.4.0-rc.3。
   - 对个人服务器：若用 Docker/Podman/K8s，这是今天最优先处理的补丁项。
   - 来源：`/root/obsidian-vault/资讯更新/服务器安全运维/2026-07-01-OpenClaw.md`

2. **OpenAI GeneBench-Pro + Anthropic Claude Science：科研 Agent 产品化加速**
   - OpenAI 推出 GeneBench-Pro 测试 Agent 研究判断力，GPT-5.6 Sol pass rate 仅 28.7%；Anthropic 发布 Claude Science 面向科研人员 workbench。
   - 对 OpenClaw：科研 Agent 方向主流化，"可审计 artifacts + 工具链 + 输出验证"已成共识设计模式。
   - 来源：`/root/obsidian-vault/资讯更新/AI与大模型/2026-07-01-OpenClaw.md`

3. **Langflow RCE（CVE-2026-33017）被用于投放矿工并横向移动**
   - Trend Micro 报告攻击者扫描暴露 Langflow 实例，通过未认证 Python 注入植入矿工，会关闭 AppArmor/UFW/iptables、删除日志、植入 cron 持久化、SSH key 横向传播。
   - 对个人服务器：所有 AI 编排/模型工具（Langflow/ComfyUI/Jupyter/Gradio/FastAPI 管理面板）默认只绑 localhost 或加反代鉴权。
   - 来源：`/root/obsidian-vault/资讯更新/技术晚间补充/2026-07-01-OpenClaw.md`

4. **Microsoft SkillOpt：把 Agent skill 文件当可训练参数 + ScarfBench 揭示 Agent 自评不可靠**
   - SkillOpt 证明自然语言技能文件可像参数一样被小步编辑 + 验证集优化，不更新模型权重。
   - ScarfBench 显示最强 Agent 在企业 Java 迁移中成功率仍低于 10%，且 Agent 自评严重高估。
   - 对 OpenClaw：Skills 可引入"验证门"防止漂移；任何 Agent 输出必须独立验证而非相信模型自评。
   - 来源：`/root/obsidian-vault/资讯更新/AI与大模型/2026-07-01-OpenClaw.md`

5. **Phantom Squatting：AI 幻觉域名正在被武器化为供应链入口**
   - Unit 42 发现 LLM 稳定生成不存在品牌域名，攻击者抢注后用于钓鱼和恶意分发；约 1.3 万个已被威胁情报标记恶意。
   - 对 Agent 安全：模型输出的链接、包名、webhook 域名不能直接作为可信输入；OpenClaw 可内建 URL/域名验证闸门。
   - 来源：`/root/obsidian-vault/资讯更新/技术晚间补充/2026-07-01-OpenClaw.md`

## 对 OpenClaw / Hermes / 个人服务器有用的 3 条

1. **容器运行时补丁优先级最高**
   - runc 修复 + Helm symlink 漏洞 （CVE-2025-53547） + NVIDIA Container Toolkit 修复 + libfuse io_uring 问题：立即核查 `runc --version`、`helm version`、`nvidia-container-toolkit --version`，确认发行版 backport 状态。

2. **轻量暴露面治理清单**
   - 检查公网暴露的 Langflow/ComfyUI/Jupyter/Gradio/FastAPI 管理面；检查 Docker API 是否无鉴权暴露；检查 SSH 出站和 cron 异常文件；检查 MFA/Conditional Access 覆盖所有云应用和客户端类型。

3. **Agent 安全基线：不可信输入应默认拒绝**
   - 结合 Phantom Squatting 和 Langflow RCE：Agent 不可自动打开模型生成的 URL、包名、webhook；工具调用应在 allowlist 内执行；外部页面/代码应视为不可信输入。

## 风险 / 机会

### 风险
- **容器逃逸风险实际可被利用**：runc 三个高危直接影响默认容器部署链路。
- **AI 工具被纳入常规攻击扫描面**：Langflow 证明 AI 框架已成为与 Docker API/Jenkins 同级别的入口。
- **AI 输出成为新供应链输入**：幻觉域名、幻觉包名会绕过传统信誉检查，开发者/Agent 自动信任的风险很高。
- **Agent 自评不可靠**：ScarfBench 明确显示 Agent 声称完成≠真实完成，独立验证不可省略。

### 机会
- **OpenClaw Skills 可引入验证驱动优化**：SkillOpt 思路适合做"小步编辑 + held-out 验证 + 可回滚"的技能管理。
- **Agent 安全护栏可产品化**：URL 验证、下载隔离、allowlist、来源检查——这些在 OpenClaw 中已有部分能力，可强化为默认策略。
- **本地模型推理性能持续改善**：Ollama 0.31 在 Apple Silicon 上通过 MLX + MTP 加速 coding-agent 任务，适合低成本试错。
