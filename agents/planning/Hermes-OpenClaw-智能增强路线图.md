# Hermes / OpenClaw 智能增强路线图

关联：[[MOC]]、[[服务器架构]]、[[agents/shared/TOOLS|共用工具与记忆规则]]、[[shared-memory/exports/shared-memory|共享记忆导出]]

## 目标

让服务器上的 Hermes 和 OpenClaw 更智能、更稳定、更好管理：

- 更会记住：能可靠召回用户偏好、服务器状态、历史故障和成功方案。
- 更会查资料：能检索 Obsidian、本地 Markdown、网页、GitHub、论文等资料源。
- 更会动手：能安全调用工具、浏览器、GitHub、MCP 和后台任务。
- 更稳定：能监控中转站、服务状态、日志异常并生成修复建议。
- 更好用：QQ 指令、Dashboard、语音、文件理解、任务队列等体验增强。

---

## 第一优先级：记忆与知识增强

### 1. 共享长期记忆增强

现状已有 `shared-memory :9400`，可继续增强为 Hermes / OpenClaw 的统一长期记忆底座。

建议能力：

- 用户偏好记忆
- 服务器状态记忆
- 历史故障记忆
- 成功修复方案记忆
- 项目结构记忆
- 长期决策记忆
- 回答前按主题自动召回相关记忆

预期效果：

- 问“上次中转怎么修的”时，可直接召回历史故障和处理方案。
- 问“OpenClaw 为什么不回消息”时，可优先检查历史 401 / 502 / 上下文膨胀等问题。
- 减少重复解释服务器结构和用户偏好。

安全边界：

- 自动健康检查、导出、索引可以做。
- 删除、禁用、合并重要记忆需要人工确认。
- 候选报告优先，不自动破坏性清理。

### 2. Obsidian 知识库 RAG

把 `/root/obsidian-vault/` 做成 Hermes / OpenClaw 的主动知识库。

建议能力：

- Markdown 全文索引
- 向量索引
- 文件标题、frontmatter、wikilink 解析
- 按主题召回相关笔记
- 回答前自动查知识库
- 把研究报告、服务器架构、故障记录纳入上下文

预期效果：

- 问“总结我的服务器架构”时，不再凭空回答，而是查 vault 后总结。
- 学习报告、巡检报告、配置说明都能被复用。
- Hermes 和 OpenClaw 共享同一套知识中枢。

### 3. Web 搜索能力

建议给 Hermes / OpenClaw 配置稳定的搜索与网页读取能力。

可接入：

- DuckDuckGo 搜索
- 网页正文抓取
- GitHub 搜索
- arXiv / 论文搜索
- YouTube transcript 获取

适用场景：

- 查最新文档
- 排查新版本 bug
- 做技术调研
- 生成学习报告
- 对比开源项目

## 第二优先级：工具与执行能力增强

### 4. MCP 工具服务器

可以新增 MCP 作为外部工具扩展层。

候选 MCP：

- GitHub MCP
- Browser / Playwright MCP
- Filesystem MCP
- SQLite / Postgres MCP
- Docker MCP
- Obsidian MCP

预期效果：

- 工具扩展更标准化。
- Hermes / OpenClaw 可像插件一样调用外部能力。
- 降低自定义脚本散落的维护成本。

### 5. 浏览器自动化

给服务器加浏览器自动化环境，用于真实页面检查。

能力：

- 打开网页
- 点击按钮
- 登录后台
- 截图
- 自动测试 Dashboard
- 检查移动端布局
- 验证按钮成功 / 失败反馈

适合场景：

- Hermes Dashboard 二开
- OpenClaw 管理 UI
- 自建服务后台测试
- 表单、按钮、移动端体验验证

### 6. GitHub 自动化增强

已有 GitHub 自动同步基础，可进一步增强工程化流程。

建议能力：

- 自动生成 PR
- 自动写 issue
- 自动 code review
- 自动检查敏感文件
- 自动分析 CI 失败
- 自动生成 changelog
- 自动同步前后差异报告

安全边界：

- 敏感文件、数据库文件、备份文件继续排除。
- 自动推送前保留脱敏流程。
- 重大改动可生成候选 PR，不直接合并。

## 第三优先级：稳定性与路由增强

### 7. 模型路由系统

目标是让不同任务走不同模型，同时保持主回复质量。

建议策略：

- 主回复默认走中转 GPT。
- 本地模型只做低风险辅助任务。
- 中转异常时生成告警，不擅自切换主路由。
- 每次路由测试标注来源。

适合本地模型的任务：

- 记忆分类
- 摘要压缩
- 日志归纳
- 文档索引
- 候选报告生成

不建议默认交给本地模型的任务：

- 复杂服务器配置
- 主 QQ 回复
- 高风险代码修改
- 多工具长链路排障

### 8. 中转站健康监控与熔断

对常用中转站做持续探活。

监控内容：

- `/models` 可用性
- `/chat/completions` 可用性
- 延迟
- 成功率
- 401 / 429 / 502 / 503 类型
- token 吞吐
- key 是否被封
- 并发上限趋势

建议策略：

- 每 5 或 10 分钟探活。
- 连续失败时 QQ 通知。
- 不自动换中转，除非明确确认。
- 对高并发压测设置保护阈值，避免再次触发 key 风控。

### 9. Hermes / OpenClaw 自愈巡检

新增安全自愈服务，优先检查，不轻易修改。

检测对象：

- Hermes gateway
- OpenClaw gateway
- QQBot WebSocket
- shared-memory
- Hermes Dashboard
- System Monitor
- Obsidian Knowledge Web
- 中转站
- cron 任务
- 关键端口
- 日志异常

动作分级：

- 轻度：只报告。
- 中度：可重启 gateway，但需要明确策略。
- 重度：生成候选修复方案，等人工确认。

## 第四优先级：管理体验增强

### 10. 统一管理 Dashboard

在现有 Hermes Dashboard 上继续增强管理能力。

候选模块：

- Hermes 配置管理
- OpenClaw 配置管理
- Provider / API key 管理
- 中转站测速
- 模型可用性探活
- 记忆检索面板
- cron 任务管理
- 服务状态面板
- 日志查看
- 一键重启 Hermes / OpenClaw
- 手机端适配

UI 要求：

- 原生集成到现有界面。
- 手机端不拥挤。
- 所有按钮有明确成功 / 失败反馈。
- 涉及密钥字段必须遮罩显示。
- 修改配置前自动备份。

### 11. 任务队列 / 后台任务中心

新增本地任务队列，承接长任务。

能力：

- 长任务不阻塞 QQ 回复
- 状态可查询
- 支持暂停、重试、取消
- 完成后 QQ 通知
- 结果保存到 Obsidian
- 失败时保存日志和恢复建议

适合任务：

- 长篇研究
- 服务器巡检
- 批量文件整理
- 多项目同步
- 学习报告生成
- 代码仓库扫描

### 12. 统一日志和可观测性

建议建立轻量日志索引，而不是只靠 `journalctl` 临时查。

可选方案：

- SQLite 日志索引
- Loki / Grafana
- Prometheus
- 自建轻量日志 Dashboard

监控内容：

- Hermes 日志
- OpenClaw 日志
- QQBot 日志
- Nginx 日志
- 中转调用日志
- cron 输出
- token 用量
- 错误类型分布

## 第五优先级：交互体验增强

### 13. 语音输入 / 语音回复

Hermes 可配置 STT / TTS。

能力：

- QQ 语音转文字
- 回复转语音
- 本地 whisper 或云端 whisper
- Edge TTS 免费语音

适用场景：

- 手机上快速发语音指令
- 长回复转语音听
- 低频但体验提升明显

### 14. 文件、图片、PDF 理解

增强多模态输入能力。

能力：

- 图片识别
- 截图分析
- PDF 解析
- Word / Excel 解析
- 长文档摘要
- 自动归档到 Obsidian

适用场景：

- 看报错截图
- 分析后台页面
- 总结 PDF 文档
- 整理表格内容

### 15. QQ 指令菜单

给 QQBot 增加常用快捷指令。

候选命令：

```text
/状态
/重启openclaw
/查中转
/查记忆 xxx
/生成日报
/服务器巡检
/今日任务
/最近错误
/压测中转
/同步github
```

预期效果：

- 减少重复输入长句。
- 常用操作标准化。
- 更适合手机端使用。

## 推荐实施路线

### 阶段 1：马上值得做

1. Obsidian 知识库 RAG
2. 中转站健康监控
3. Hermes / OpenClaw 自愈巡检
4. Dashboard 增强管理页

### 阶段 2：进一步增强智能

1. 共享记忆自动召回
2. Web 搜索 / 浏览器自动化
3. GitHub 自动 PR / issue / review
4. 任务队列中心

### 阶段 3：高级体验

1. QQ 指令菜单
2. 语音输入输出
3. 图片 / PDF / 网页理解
4. 多 Agent 协作工作流

---

## 最推荐先做的一项

优先做：

> Obsidian + shared-memory + Hermes / OpenClaw 自动召回

原因：

- 最直接提升“智能感”。
- 能让 Hermes / OpenClaw 更懂服务器、历史故障和用户偏好。
- 能减少重复解释、减少幻觉。
- 对后续 Dashboard、巡检、自愈、研究报告都有基础价值。

---

## 安全原则

1. 不自动删除、合并重要记忆或 skills。
2. 修改 Hermes / OpenClaw 配置前保留可恢复备份。
3. Provider / API key 字段必须遮罩显示。
4. 中转站压测限制默认并发，避免触发 key 风控。
5. 学习调研任务只生成报告，不主动改服务器代码或配置。
6. 服务器配置类任务必须验证服务存活和端口状态。
