# 数据科学基础与工程衔接

> 关联：[[软件工程/数据库设计与性能优化|数据库设计与性能优化]]、[[软件工程/现代软件工程实战总纲|现代软件工程实战总纲]]

## 1. 数据科学位置

数据科学与软件工程结合形成看板、行为分析、推荐、风险识别、预测、异常检测、LLM 数据处理。失败常因数据质量差、指标混乱、训练/生产不一致、缺少监控，而不是模型不够高级。

## 2. 流程

问题定义 → 数据收集 → 清洗 → EDA → 特征工程 → 建模 → 评估 → 部署 → 监控 → 迭代。

## 3. 问题定义

先问业务问题、使用者、错误成本、实时性、解释性、数据可得性、成功指标。不要一开始问用什么模型。

## 4. 数据质量

检查缺失、重复、异常值、时间戳、单位、类别拼写、标签延迟、数据泄漏、样本偏差。每个数据集应有数据质量报告。

## 5. 特征工程

特征必须预测时可获得，训练和线上计算一致，避免数据泄漏，记录特征定义，监控分布漂移。

## 6. 评估

分类用 precision/recall/F1/ROC-AUC/PR-AUC，回归用 MAE/RMSE/MAPE。离线指标不等于业务成功，还要看可解释性、执行成本和用户反馈。

## 7. 工程化

Notebook 适合探索，生产需要模块化：data、features、models、evaluation、api。记录数据版本、代码版本、参数、随机种子、依赖、指标、模型路径。

## 8. 模型监控

监控请求量、延迟、错误率、输入分布、输出分布、数据漂移、模型效果、人工反馈、业务指标。现实变化会让模型退化。
