# 数据库设计与性能优化

> 关联：[[软件工程/数据库迁移与数据治理|数据库迁移与数据治理]]、[[软件工程/数据库索引案例库|数据库索引案例库]]

## 1. 目标

数据库设计要表达业务、保证一致性、支持查询、可迁移、可审计。数据库比代码更难重构，早期设计要比普通模块更谨慎。

## 2. 建模

先识别实体、关系、状态、事件，再设计表。核心表应有主键、时间字段、约束、索引、注释和变更记录。

## 3. 主键与外部 ID

内部主键可以自增，外部暴露可用 UUID/public_id，避免 ID 枚举。业务编码不要当主键。

## 4. 约束

NOT NULL、UNIQUE、FOREIGN KEY、CHECK 是最后防线。邮箱唯一、金额非负、状态枚举都应有数据库约束。

## 5. 索引

索引用于查询条件、排序、唯一和外键。组合索引顺序取决于查询模式。索引会降低写入、占空间，必须用 EXPLAIN ANALYZE 验证。

## 6. 查询优化

避免 N+1；列表必须分页；大 offset 需要游标分页；只取需要字段；慢查询要看执行计划和扫描行数。

## 7. 事务与并发

事务要短。高并发扣库存、支付回调、任务消费需要锁、唯一约束、幂等表和状态机。

## 8. 迁移

采用 expand-contract：先加字段/新表，双写或兼容读，回填，切换，最后删除旧结构。大表变更要评估锁和回滚。

## 9. 数据治理

记录字段含义、owner、质量规则、敏感等级、保留期限、备份策略。数据质量问题要像 bug 一样追踪。
