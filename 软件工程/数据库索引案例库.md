# 数据库索引案例库

> 关联：[[软件工程/数据库设计与性能优化|数据库设计与性能优化]]

## 常见索引

外键：`CREATE INDEX idx_orders_user_id ON orders(user_id);`

状态+时间：`CREATE INDEX idx_orders_status_created ON orders(status, created_at DESC);`

唯一：`CREATE UNIQUE INDEX idx_users_email ON users(email);`

## 判断

用 `EXPLAIN ANALYZE` 验证，不凭感觉加。索引会降低写入速度，占磁盘。
