# 日志错误处理规范

> 关联：[[运维/可观测性与日志体系|可观测性与日志体系]]、[[软件工程/API 设计规范与错误码|API 设计规范与错误码]]

## 1. 错误层次

用户错误、业务错误、权限错误、外部依赖错误、系统错误、未知错误。不同错误对应不同 HTTP 状态码和日志级别。

## 2. 日志字段

request_id、user_id、service、route、status、duration_ms、error_code、message、stack、resource_id。

## 3. 不记录

密码、token、cookie、Authorization、密钥、证书、个人敏感信息、完整请求体中的隐私字段。

## 4. 错误响应

对用户友好，对开发可追踪。返回 request_id，不直接暴露堆栈。
