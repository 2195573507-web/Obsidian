# 代码审查清单

> 关联：[[软件工程/测试与质量保障体系|测试与质量保障体系]]、[[软件工程/现代软件工程实战总纲|现代软件工程实战总纲]]

## 1. 审查维度

需求是否满足、边界是否覆盖、权限是否正确、错误处理是否清晰、日志是否足够、测试是否存在、性能是否可接受、安全是否有漏洞、文档是否更新。

## 2. 安全重点

认证授权、IDOR、SQL 注入、XSS、CSRF、文件上传、路径穿越、密钥泄露、日志泄露、依赖漏洞。

## 3. 可维护性

命名、分层、重复代码、函数长度、抽象是否过度、是否破坏模块边界、是否引入全局状态。

## 4. 审查结论模板

```markdown
## 总体结论
- 风险等级：低/中/高
## 必须修改
## 建议修改
## 可后续处理
## 已验证
```
