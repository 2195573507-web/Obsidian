# 后台管理系统设计

> 关联：[[软件工程/原生前端与 UIUX 产品化实践|原生前端与 UIUX 产品化实践]]、[[软件工程/权限系统 RBAC 与 ABAC|权限系统 RBAC 与 ABAC]]

## 1. 后台的核心任务

后台不是页面集合，而是管理操作系统。核心能力：搜索、筛选、列表、详情、编辑、批量操作、权限、审计、导入导出、任务状态、错误反馈。

## 2. 信息架构

导航按用户任务组织，不按数据库表机械组织。常见模块：概览、用户、内容、订单、系统、日志、配置、任务。

## 3. 列表页

必须支持分页、搜索、筛选、排序、状态标签、空状态、错误状态、批量操作。危险操作隐藏在更多菜单并二次确认。

## 4. 表单

字段分组、默认值、实时校验、保存反馈、离开未保存提醒。错误应指向具体字段。

## 5. 审计与回滚

后台高风险操作必须有审计日志。可逆操作优先软删除、禁用、撤销，而不是直接物理删除。

## 6. 移动端

管理后台移动端不应拥挤。核心操作保留，复杂表格横滚或卡片化，按钮触控面积足够。
