# 测试与质量保障体系

> 关联：[[软件工程/现代软件工程实战总纲|现代软件工程实战总纲]]、[[软件工程/FastAPI 后端工程实践|FastAPI 后端工程实践]]

## 1. 测试目的

测试不是证明没有 bug，而是快速发现错误、防止回归、固化业务规则、支撑重构、降低上线风险。

## 2. 测试金字塔

单元测试快而稳定，适合业务规则；集成测试验证 API+DB、Service+Repository；E2E 测试覆盖关键用户流程但成本高。

## 3. TDD

RED：先写失败测试；GREEN：最少代码通过；REFACTOR：测试保护下重构。TDD 的价值是先设计行为和边界。

## 4. 后端测试

Service 测业务规则；API 测契约；权限测试必须覆盖；数据库测试检查约束、事务、迁移、并发边界。

## 5. 前端测试

即使原生 JS，也可测试数据转换、表单校验、URL 参数、状态变化、HTML escape、API 错误处理。复杂时用 Playwright/Vitest。

## 6. 质量保障

质量还包括 code review、lint、类型检查、安全扫描、依赖检查、架构评审、文档审查、灰度、监控和复盘。

## 7. CI/CD

CI：安装依赖、格式、lint、类型、测试、安全扫描、构建。CD：部署 staging、冒烟、确认、生产、健康检查、观察。上线前必须知道如何回滚代码、配置和数据库迁移。
