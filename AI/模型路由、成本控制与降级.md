# 模型路由、成本控制与降级

> 关联：[[AI/多 Agent 协作、安全评测与生产运维|多 Agent 协作、安全评测与生产运维]]、[[AI/LLM 评测体系与黄金测试集|LLM 评测体系与黄金测试集]]

## 1. 模型路由的目标

不同任务不需要同一个模型。路由目标：在质量、成本、延迟、可用性、安全之间动态选择最合适模型或链路。

## 2. 路由维度

| 维度 | 示例 |
|---|---|
| 任务复杂度 | 简单问答、代码修复、长研究 |
| 风险等级 | 只读、写操作、生产变更 |
| 上下文长度 | 短对话、长文档、全仓库 |
| 工具需求 | 无工具、读工具、写工具 |
| 质量要求 | 草稿、最终报告、关键决策 |
| 成本预算 | 免费/低价/高质量 |
| 延迟要求 | 即时、后台、cron |

## 3. 路由策略

简单任务用便宜快模型；复杂推理、工具排障、安全决策用强模型；长文档可先小模型摘要再强模型综合；多 agent 子任务可混用模型。

## 4. 成本控制

- 上下文压缩；
- RAG top_k 调优；
- 工具输出裁剪；
- 缓存检索和中间摘要；
- 小模型预分类；
- 批量处理 embedding；
- 限制多 Agent 数量；
- 为 cron 设置预算；
- 记录 cost per task。

## 5. 降级

| 故障 | 降级 |
|---|---|
| 主模型不可用 | 备用模型 |
| RAG 不可用 | 说明无法访问知识库，只回答通用部分 |
| 工具失败 | 给手动命令或稍后重试 |
| 成本超限 | 停止扩写，保存进度 |
| 延迟过高 | 后台任务/cron |
| 安全不确定 | 转人工确认 |

## 6. 监控指标

按模型记录：成功率、超时率、平均 token、P95 延迟、平均成本、schema 失败率、工具失败率、用户重试率、人工接管率。
