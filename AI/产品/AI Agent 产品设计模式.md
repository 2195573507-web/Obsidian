# AI Agent 产品设计模式

> 关联：[[软件工程/后台管理系统设计|后台管理系统设计]]、[[AI/Agent 安全攻防与权限设计|Agent 安全攻防与权限设计]]

## 1. Agent 产品不是聊天框

聊天只是入口。Agent 产品需要任务状态、工具记录、审批、结果验证、回滚、历史、权限和反馈。

## 2. 核心界面模块

- 任务目标输入；
- 计划预览；
- 工具调用时间线；
- 风险提示；
- 人工确认卡片；
- 结果和证据；
- 失败恢复；
- 反馈按钮；
- 成本和耗时。

## 3. UX 原则

用户应知道 Agent 正在做什么、为什么要做、用了什么工具、是否产生副作用、下一步是否需要确认。

## 4. 高风险操作设计

用“候选方案 → diff/影响范围 → 确认 → 执行 → 验证 → 回滚入口”的流程，不允许一句话直接删改生产。

## 5. 产品指标

任务完成率、人工介入率、平均步骤数、失败恢复率、用户撤销率、成本/任务、满意度、安全拦截次数。
