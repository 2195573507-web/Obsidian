# 事件埋点规范

> 关联：[[数据/指标体系设计|指标体系设计]]

## 1. 事件结构

```json
{"event":"button_clicked","user_id":"u1","timestamp":"...","properties":{"page":"dashboard","button":"save"}}
```

## 2. 命名

事件用过去式或动作名：page_viewed、form_submitted、task_created。属性名稳定，不随 UI 文案改变。

## 3. 必备字段

event、timestamp、user/session、page、source、version、request_id。

## 4. 质量

埋点要有文档、测试、去重、隐私过滤、版本管理。
